# Инфраструктура проекта

## Архитектура решения
[Сырые CSV] → [Yandex Cloud Object Storage] → [Yandex Query] → [Агрегированные данные] → [Yandex DataLens] → [Дашборды]

## Компоненты Yandex Cloud

### 1. Yandex Object Storage (S3-совместимое)
- **Бакет:** `online-school-analytics`
- **Структура папок:**
  - `raw/` - исходные CSV файлы (таблицы campaigns, clicks, leads и т.д.)
  - `aggregated/` - результаты агрегаций:
    - `sales_funnel/` - данные для воронки продаж
    - `retention_report/` - данные для когортного анализа

### 2. Yandex Query (YQL)
- Использовался для трансформации данных
- Особенности:
  - Внешние таблицы, привязанные к файлам в Object Storage
  - Сложные запросы с JOIN 5+ таблиц
  - Материализация результатов для ускорения работы дашбордов

### 3. Yandex DataLens
- Подключение к агрегированным данным в Object Storage
- Создание датасетов с вычисляемыми полями
- Настройка интерактивных чартов и дашбордов

## Процесс работы
1. Загрузка данных в Object Storage через консоль управления
2. Написание и отладка YQL-запросов в Yandex Query
3. Материализация агрегированных данных обратно в Object Storage
4. Создание подключений и датасетов в DataLens
5. Разработка дашбордов с чартами и селекторами
