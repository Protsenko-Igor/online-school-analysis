# Описание данных

## Источники данных
Данные представляют собой эмуляцию реальных данных онлайн-школы, которые могли бы собираться из следующих источников:
1. **Рекламные системы** (Яндекс.Директ, VK Реклама и др) → таблицы `campaigns`, `clicks`
2. **CRM-система** (например, amoCRM) → таблицы `leads`, `interactions`, `sales`
3. **LMS (Learning Management System)** → таблица `student_activity`
4. **Сквозная аналитика** (Google Analytics, Яндекс.Метрика) → атрибуция `utm_source`, `utm_medium`

## Структура таблиц

### 1. Маркетинг и продажи
| Таблица | Описание | Ключевые поля |
|---------|----------|---------------|
| `campaigns` | Рекламные кампании | `campaign_id`, `source`, `budget` |
| `clicks` | Клики по рекламе с атрибуцией | `click_id`, `campaign_id`, `cost` |
| `leads` | Заявки от потенциальных клиентов | `lead_id`, `user_id`, `created_at` |
| `interactions` | Взаимодействия с лидами | `type` (звонок, встреча), `status` |
| `sales` | Совершенные продажи | `sale_id`, `product_name`, `amount` |

### 2. Обучение и вовлеченность
| Таблица | Описание | Ключевые поля |
|---------|----------|---------------|
| `student_activity` | Активность студентов после покупки | `activity_date`, `lessons_completed`, `homework_submitted` |

## Связи между таблицами
Основные связи осуществляются через:
- `campaign_id` → от клика до продажи (атрибуция)
- `lead_id` → от заявки до взаимодействий и продажи
- `user_id` → сквозная идентификация пользователя

## Где хранятся данные
В рамках проекта данные были загружены в **Yandex Cloud Object Storage** в бакет `online-school-analytics` со следующей структурой:
```
online-school-analytics/
├── raw/
│   ├── campaigns.csv
│   ├── clicks.csv
│   ├── leads.csv
│   └── ...
└── aggregated/
    ├── sales_funnel/
    └── retention_report/
```
